# Academic Website for Faniry Razakason

This is a lightweight static academic website built with plain HTML, CSS, and JavaScript so it is easy to edit and easy to host for free.

## Files to edit

- `index.html`: Main content, bio, research sections, contact details, profile links
- `script.js`: Publications list
- `style.css`: Design and layout

## Personal info placeholders

Search for `TODO:` in the project to find every place where you should replace sample content with your own details.

## Add your CV

1. Create a folder named `files`
2. Put your PDF inside it
3. Rename the file to `Faniry_Razakason_CV.pdf`

Or update the CV link in `index.html` to match your preferred filename.

## Free hosting options

### Option 1: GitHub Pages

Best if you want a simple, professional workflow tied to a GitHub repository.

1. Create a new public repository on GitHub
2. Upload these files to the repository root
3. In GitHub, open `Settings` -> `Pages`
4. Under `Build and deployment`, choose `Deploy from a branch`
5. Select your main branch and the `/ (root)` folder
6. Save, then wait for the site to publish

Your website will usually appear at:

`https://yourusername.github.io/repository-name/`

If you name the repository `yourusername.github.io`, the site can appear at:

`https://yourusername.github.io/`

### Option 2: Netlify

Best if you want the fastest no-code deployment.

1. Go to Netlify
2. Drag your project folder into the deploy area
3. Netlify will instantly give you a free public URL

You can later connect the same site to GitHub for automatic updates.

### Option 3: Cloudflare Pages

Best if you want another strong free static hosting option with Git integration.

1. Push the project to GitHub
2. In Cloudflare Pages, create a new Pages project
3. Connect your GitHub repository
4. Because this is a plain static site, leave the build command empty
5. Set the output directory to `/`
6. Deploy

## Recommended path

For an academic homepage, GitHub Pages is usually the simplest long-term choice:

- free
- reliable
- easy to update
- good enough for a personal academic site
- works well with a custom domain later if you want one
